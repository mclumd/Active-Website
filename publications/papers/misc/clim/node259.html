Frame Manager Operations

frame-manager frame

Returns the current frame manager of frame if it is adopted, otherwise returns nil.

(setf frame-manager) manager frame

Changes the frame manager of frame to manager. Effectively, the frame is disowned from its old frame manager and is adopted into the new frame manager. Transferring a frame preserves its frame-state, for example, if the frame was previously enabled it will be enabled in the new frame manager.

frame-state frame

Returns one of :disowned, :enabled, :disabled, or :shrunk indicating the current state of frame.

adopt-frame frame-manager frame disown-frame frame-manager frame

These functions insert or remove a frame from a frame manager's control. These functions allow a frame manager to allocate and deallocate resources associated with a frame. For example, removing a frame from a frame manager that is talking to a remote server allows it to release all remote resources used by the frame.

frame-pane frame pane-frame pane

These functions return the pane of the frame and the frame of the pane, respectively. pane-frame can be invoked on any pane in a frame's pane hierarchy, not just its single root pane.

enable-frame frame disable-frame frame shrink-frame frame

These functions force a frame into the enabled, disabled, or shrunken states. A frame in the enabled state may be visible if it is not occluded or placed out of the user's focus of attention. A disabled frame is never visible. A shrunk frame is accessible to the user for re-enabling, but may be represented in some abbreviated form, such as an icon or a menu item.

generate-panes frame frame-manager

This function is invoked by a standard method of adopt-frame. It is the responsibility of the frame implementor to provide a method that invokes setf of frame-pane on the frame with a value of type pane. define-application-frame automatically supplies a generate-panes method if the :panes option is used.

layout-frame frame &optional width height

Resizes the frame and layouts out the current pane hierarchy according to the layout protocol. The basics of the layout protocols are described in Section 29.3. This function is automatically invoked on a frame when it's adopted, after its pane hierarchy is generated.

If width and height are provided, then this function resizes the frame to this size. It is an error to provide just width.

If no optional arguments are provided, this function resizes the frame to the preferred size of the top-level pane as determined by the space composition pass of the layout protocol.

In either case, after the frame is resized, the space allocation pass of the layout protocol is invoked on the top-level pane.

reconfigure-frame frame pane-name &key resize-frame

Switches the top-level pane to the pane specified by pane-name. This function can only be used in conjunction with the :panes options to define-application-frame, which is used to name panes.

If resize-frame is nil, then the frame is not resized to accommodate the new pane hierarchy.

frame-named-panes frame

Returns the list of possible top-level pane names. This is the set of valid inputs to the pane-name argument of reconfigure-frame.

frame-pane-name frame

Returns the name of the current top-level pane of the frame.

find-pane-for-frame frame frame-manager

This function is invoked by a standard method of adopt-frame. It must return the root pane of the frame's layout. It is the responsibility of the frame implementor to provide a method that constructs the frame's top-level pane. define-application-frame automatically supplies a a method for this function if the :panes option is used.

frame-properties frame property (setf frame-properties) new-value frame property

Frame properties can be used to associate frame specific data with frames without adding additional slots to the frame's class. However, for important properties, the frame developer want to add slots to store highly utilized information.

note-command-enabled frame-manager frame command-name enabled

Notifies the frame manager frame-manager that the command named by command-name has been enabled or disabled in the frame frame. enabled is a boolean that indicates whether the command was enabled or disabled. The frame manager can update the appearance of the user interface as appropriate, for instance, by ``graying out'' a newly disabled command from a command menu or menu bar.
