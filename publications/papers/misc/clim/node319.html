Presentation Types

  • The argument list for with-output-as-presentation has been changed to make stream, object, and type be required arguments instead of keyword arguments. This is because it is always necessary to supply those arguments in order for with-output-as-presentation to work. Compability code will be provided to support the old syntax.

  • The activation-characters, additional-activation-characters, blip-characters, and additional-blip-characters keyword arguments to the accept functions have been renamed to activation-gestures, additional-activation-gestures, blip-gestures, and additional-blip-gestures. Compability code will be provided to support the old keyword arguments.

  • The arglists for presentation translators and their documentation and tester components have been changed to take a single position object argument and a list of named (keyword) arguments. Except for translators that omit the object argument or have it in other than the initial position of the arglist, this will not pose a problem. This change can be detected.

  • The shift-mask keyword argument to test-presentation-translator, find-applicable-translators, presentation-matches-context-type, and find-innermost-applicable-presentation has been renamed to modifier-state in order to be consistent with the device event terminology. Compability code will be provided to support the old keyword.

  • The shifts keyword argument to define-gesture-name has been renamed to modifiers in order to be consistent with the device event terminology. Compability code will be provided to support the old keyword.

  • dialog-view and +dialog-view+ have been renamed to textual-dialog-view and +textual-dialog-view+ in order to accurately reflect what they are. Likewise, menu-view and +menu-view+ have been renamed to textual-menu-view and +textual-menu-view+. Compatibility classes and constants will be provided.
