Predefined Gadgets

CLIM provides a standard set of adaptive gadgets. This section documents the predefined gadget types. Note that not all operations will necessarily be generated by particular toolkit implementations. For example, a user interface toolkit that is designed for a 3-button mouse may generate significantly more gadget events than one designed for a 1-button mouse. More on this below.

push-button

The push button gadget provides press-to-activate switch behavior.

:label--string

Specifies the label associated with the push button.

:label-text-style--text-style

Specifies the text style that will be used when drawing the label.

:show-as-default-p--boolean

When show-as-default-p is true, the push button will be draw with a heavy border, which indicates that this button is the ``default operation''.

button-label button

Returns the label of the button.

(setf button-label) label button

Changes the label of button to the new label string label.

button-text-style button

Returns the text style of the button's label.

(setf button-text-style) text-style button

Changes the text style of button's label to the new text style text-style.

push-button-show-as-default-p push-button

Returns true if the push button will be shown as the default, otherwise returns false.

(setf push-button-show-as-default-p) state push-button

Changes the ``show as default'' state of push-button to state, which must be t or nil.

value-change-callback button client gadget-id value

Called when the toolkit determines that the button has been pushed. The value of value is indeterminate for push buttons, and should be ignored.

arm-callback push-button client gadget-id

Called when the pointer is clicked on the push button gadget identified by gadget-id, preparatory to a value change. (The exact time when this is called may vary slightly from one toolkit to another.)

disarm-callback push-button client gadget-id

Called when the pointer button is released after an arm-callback. (The exact time when this is called may vary slightly from one toolkit to another.)

The normal sequence of callbacks is arm-callback, then value-change-callback, then disarm-callback.

toggle-button

The toggle button gadget provides ``on/off'' switch behavior. This gadget typically appears as a box that is optionally highlighted with a check-mark. If the check-mark is present, the gadget's value is t, otherwise it is nil.

:label--string

Specifies the label associated with the toggle button.

:label-text-style--text-style

Specifies the text style that will be used when drawing the label.

:indicator-type--:one-of or :some-of

The indicator-type controls the appearance of the toggle button. For example, many toolkits present a one-of-many choice differently from a some-of-many choice.

button-label button

Returns the label of the button.

(setf button-label) label button

Changes the label of button to the new label string label.

button-text-style button

Returns the text style of the button's label.

(setf button-text-style) text-style button

Changes the text style of button's label to the new text style text-style.

toggle-button-indicator-type toggle-button

Returns the indicator type of toggle-button.

(setf toggle-button-indicator-type) type toggle-button

Changes the indicator type of toggle-button to type, which must be :one-of or :some-of.

value-change-callback toggle-button client gadget-id value

Called when the toolkit determines that the toggle button has been toggled. The value of value is indeterminate for toggle buttons, and should be ignored.

arm-callback toggle-button client gadget-id

Called when the pointer is clicked on the toggle button gadget identified by gadget-id, preparatory to a value change. (The exact time when this is called may vary slightly from one toolkit to another.)

disarm-callback toggle-button client gadget-id

Called when the pointer button is released after an arm-callback. (The exact time when this is called may vary slightly from one toolkit to another.)

The normal sequence of callbacks is arm-callback, then value-change-callback, then disarm-callback.

scroll-bar

:orientation--:horizontal or :vertical

Specifies the orientation of the scroll bar, either horizontal or vertical.

:min-value--real number, default 0

Specifies the minimum value of the scroll bar.

:max-value--real number, default 1

Specifies the maximum value of the scroll bar.

:unit-increment--real number, default 1

Specifies the increment between tick marks on the scroll bar.

:page-increment--real number, default 1

scroll-bar-orientation scroll-bar

Returns the orientation of the scroll bar.

(setf scroll-bar-orientation) orientation scroll-bar

Changes the orientation of the scroll bar to orientation.

scroll-bar-min-value scroll-bar

Returns the minimum value of the scroll bar.

(setf scroll-bar-min-value) value scroll-bar

Changes the minimum value of the scroll bar to value.

scroll-bar-max-value scroll-bar

Returns the minimum value of the scroll bar.

(setf scroll-bar-max-value) scroll-bar

Changes the maximum value of the scroll bar to value.

scroll-bar-unit-increment scroll-bar

(setf scroll-bar-unit-increment) scroll-bar

scroll-bar-page-increment scroll-bar

(setf scroll-bar-page-increment) scroll-bar

drag-callback scroll-bar client gadget-id value

Invoked each time the ``thumb'' of the scroll bar is moved while dragging the pointer.

value-change-callback scroll-bar client gadget-id value

Invoked when a pointer button is pressed a some point in the scroll bar.

All of the following callbacks are invoked when appropriate parts of the scroll bar are clicked on. Note that each implementation may not have ``hot spots'' corresponding to each of these generic operations.

scroll-to-top-callback scroll-bar client gadget-id scroll-to-bottom-callback scroll-bar client gadget-id scroll-up-line-callback scroll-bar client gadget-id scroll-up-page-callback scroll-bar client gadget-id scroll-down-line-callback scroll-bar client gadget-id scroll-down-page-callback scroll-bar client gadget-id

value-change-callback is invoked when the indicator is released after dragging it.

slider

:orientation--:horizontal or :vertical

:min-value--number, default 0

:max-value--number, default 1

:show-value-p--boolean, default NIL.

:show-value-p controls whether or not a numeric display corresponding to the slider's value is visible.

slider-orientation slider

Returns the orientation of the slider.

(setf slider-orientation) orientation slider

Changes the orientation of the slider to orientation.

slider-min-value slider

Returns the minimum value of the slider.

(setf slider-min-value) value slider

Changes the minimum value of the slider to value.

slider-max-value slider

Returns the minimum value of the slider.

(setf slider-max-value) slider

Changes the maximum value of the slider to value.

slider-show-value-p slider

(setf slider-show-value-p) slider

drag-callback slider client gadget-id value

Invoked each time the ``thumb'' of the slider is moved while dragging the pointer.

value-change-callback slider client gadget-id value

Invoked when a pointer button is pressed a some point in the slider.

value-change-callback is invoked when the indicator is released after dragging it.

radio-box

:current-selection

radio-box-current-selection gadget (setf radio-box-current-selection) selection gadget

value-change-callback radio-box client gadget-id value

value is the newly-selected button gadget.

The radio box enforces selection behavior like car radio buttons. Only one of the buttons managed by the radio box may be ``on'' at any time. As the current selection changes, the previously selected button and the newly selected button both have their value-change-callback handlers invoked. The radio box gadget can manage any other gadget whose gadget-value is constrained to be a boolean. At present, only the toggle button satisfies this criterion. radio-box-current-selection is the programmer interface to determining which of the gadgets is selected.

Note that the radio box is the client of its buttons.

cascade-button

:menu-group--a menu group

cascade-button-menu-group gadget (setf cascade-button-menu-group) gadget

None

When a cascade button is clicked upon, a menu containing its menu group.

Note that the specific user interface to cascade buttons will differ in each underlying toolkit. For example, in some toolkits, merely dragging the mouse across a cascade button is enough to cause its submenu to become visible. In other implementations, the mouse may have to leave the cascade button through a particular boundary. In others, a click may be necessary.
