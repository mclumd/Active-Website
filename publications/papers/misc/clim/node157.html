Buffering of Output

Some sheets that support the output protocol may buffer output. When buffering is enabled on a sheet, the time at which output is actually done on the sheet is unpredictable. force-output or finish-output can be used to ensure that all pending output gets completed. If the sheet is a bidirectional stream, a force-output is performed whenever any sort of input is requested on the stream.

with-buffered-output provides a way to control when buffering is enabled on a sheet. By default, CLIM's interactive streams are buffered if the underlying window system supports buffering.

sheet-buffering-output-p sheet

Returns true if the sheet sheet is currently buffering output, otherwise returns false.

(setf sheet-buffering-output-p) buffer-p sheet

Sets sheet-buffering-output-p of the sheet sheet to buffer-p.

with-output-buffered (sheet &optional (buffer-p t)) &body body

If buffer-p is true (the default), this causes the sheet designated by sheet to start buffering output, and evaluates body in that context. If buffer-p is false, force-output will be called before body is evaluated. When body is exited (or aborted from), force-output will be called if output buffering will be disabled after with-output-buffered is exited.

The sheet argument is not evaluated, and must be a symbol that is bound to a sheet. If sheet is t, *standard-output* is used. body may have zero or more declarations as its first forms.
