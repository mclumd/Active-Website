Overview

Gadget panes are panes that implement such common toolkit components as push buttons or scroll bars. Each gadget pane class has a set of associated generic functions that serve the role of ``callbacks'' in ``traditional'' toolkits. For example, a push-button has a ``value change'' callback function that is invoked when its button is ``pressed''.

Moon Doing gadgets as panes seems dangerous, since I thought panes are statically laid out but one should be able to draw a knob anywhere in the drawing plane and have it immediately start working. I would have thought of gadgets as presentations rather than panes.

The gadget pane definitions supplied here are abstract. That is, the gadget definition does not specify the specific user interface of the gadget pane. In other words, it is not defined whether the user clicks on the button with the mouse or moves the mouse over the button and presses some key on the keyboard to invoke the ``value change'' callback. Particular toolkit implementations will specify the ``look and feel'' of their gadgets.

Each of the abstract gadget panes has at least one implementation, written in Lisp using the CLIM input/output kernel capabilities. Its appearance is achieved via calls to the CLIM graphics functions, its interactive behavior defined in terms of the CLIM input event processing mechanism. Since these gadget implementations are written entirely in terms of CLIM, they are portable across all supported CLIM host window systems. However, since the specific look and feel of each such gadget is ``fixed'' in CLIM Lisp code, the gadget implementation will look and behave the same in all environments.
