General Regions

A region is an object that denotes a set of mathematical points in the plane. Regions include their boundaries, that is, they are closed. Regions have infinite resolution.

A bounded region is a region that contains at least one point and for which there exists a number, 3#3, called the region's diameter, such that if 4#4 and 5#5 are points in the region, the distance between 4#4 and 5#5 is always less than or equal to 3#3.

An unbounded region either contains no points or contains points arbitrarily far apart.

Another way to describe a region is that it maps every 6#6 pair into either true or false (meaning member or not a member, respectively, of the region). Later, in Chapter 14, we will generalize a region to something called a design that maps every point 6#6 into color and opacity values.

region

The protocol class that corresponds to a set of points. This includes both bounded and unbounded regions. a region region

There is no general constructor called make-region because of the impossibility of a uniform way to specify the arguments to such a function.

regionp object

Returns true if object is a region, otherwise returns false.

path

The protocol class path denotes bounded regions that have dimensionality 1 (that is, have length). It is a subclass of region and bounding-rectangle. a path path

Constructing a path object with no length (via make-line*, for example) canonicalizes it to +nowhere+.

Some rendering models support the constructing of areas by filling a closed path. In this case, the path needs a direction associated with it. Since CLIM does not currently support the path-filling model, paths are directionless.

pathp object

Returns true if object is a path, otherwise returns false.

area

The protocol class area denotes bounded regions that have dimensionality 2 (that is, have area). It is a subclass of region and bounding-rectangle. an area area

Constructing an area object with no area (via make-rectangle*, for example) canonicalizes it to +nowhere+.

areap object

Returns true if object is an area, otherwise returns false.

+everywhere+ +nowhere+

+everywhere+ is the region that includes all the points on the two-dimensional infinite drawing plane. +nowhere+ is the empty region, the opposite of +everywhere+.
