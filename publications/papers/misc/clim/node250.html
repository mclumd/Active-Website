Application Frames

SWM Integrate the CLIM 0.9 stuff, but try to address the issues that DCPL raises and all of the stuff in the mail as well.

DCPL The Silica-based macrology for defining panes and layouts doesn't easily allow, if it allows at all, for sharing panes between multiple layouts. CLIM 1.0 seems to have this capability, but the syntax for the constraints is a bit baroque. Perhaps there just isn't a good one, since in all these years we've never found it. Another way to say all of this is that the functional aspects of frames (panes and streams) needs to be separated from the layout description.

DCPL We need to decide how much of the pane, stream and layout functionality we want to present in the API and/or SPEC, and how to present it. For example, CLIM 0.9's frame-pane isn't conceptually elegant because that is how you get the current layout, and the layout is a pane, and you have to know that. Better is frame-current-layout and then have layout-XXX functions to interrogate the layout. I think CLIM 1.0 does better here.

DCPL CLIM 0.9's method of having panes names be slot names is poor because it is implicit, and problematic because it uses slot-unbound, which while it may work, shoots up a red flag. DW and presumably CLIM 1.0 have ways to ask for the pane names and ways to retrieve a pane from a frame given its name. I think this is the right model. The tricky thing, which Silica may allow and that we probably want to preserve if possible, is for a named pane to have another named pane as one of its descendants. Also have a named pane have a named stream as one of its descendants (in the event we have streams that aren't panes). This is important for viewport panes pointing at streams.

DCPL The concept of a frame manager is an important one, as it is the entity that bridges the gap between the application and the host windowing system.

DCPL The concept of server paths from CLIM 0.9 is superior to the explicit open-root-window paradigm of CLIM 1.0. CLIM 0.9 has some problems in this regard, but I have some saved mail from an ILA conversation a while that goes a long way toward removing those problems.
