Dialogue System

Conversational Adequacy

One major ongoing application of active logics is that of building a "conversationally adequate" dialogue agent (see paper on Conversational Adequacy -- one can view such an agent as having the ability to learn in McCarthy's sense of advice-taking, via conversation).

The long-range research plan envisions the following steps:

  1. Process a few simple queries via voice input, requiring the use of negative introspection, in the context of the Rochester TRAINS system rewritten to employ active logic;
  2. Handle apparent contradictions as in "Send the Boston train to NYC... [here the system does something]... No, send the Boston train to NYC" where the system has misunderstood the first request and so the user says "no" and repeats the same request. This requires the system to recognize that "no" refers to the systems response rather than to the user's first request;
  3. Understand and generate spelling in metadialogue ("The word I am saying is spelled 'xyz'...");
  4. Definitions and/or definitional queries ("What does the word 'xyz' mean?");
  5. More general metadialogue: You said, I said, you meant, I meant, etc.;
  6. Reasoning about the physical and social worlds;
  7. Focus, attention and memory;
  8. Conversational adequacy.

Implementation

  • Alma-Carne

    Alma is an active logic execution system. The major parts are the database and a set of inference rules. The database evolves through discrete states by applications of the inference rules and addition of information from external sources through the Carne process. This process repeats indefinitely.

    A ``theory'' of the domain that is being reasoned about can be loaded into Alma when it is started and the external information can be added by separate processes or by the user at the interface. The contents of the database can be viewed at the interface. The evolution of the database can be recorded in a history file. We should have a query function too.

    The Alma language has terms corresponding to the current time and the times at which formulas were derived. Formulas in the language can also be used as terms. A built in inference rule detects direct contradictions in the Alma database, removes them and asserts that there has been a contradiction. This enables the user to reason about inconsistencies in the database without the consequences of the contradiction spoiling the whole database.

    Alma together with Carne can be used to specify the behavior of an agent that can reason about and in a dynamic environment using the theory in Alma and act and observe that environment using the procedures in Carne.
