Some Alma examples

Alma (and active logic in general) is a versatile reasoning tool that has been applied to many problems. This page illustrates some of these applications by means of episodes of reasoning in Alma.

  • What is there to view.
  • How to view the examples.
  • Time situated reasoning

  • Nonmonotonic reasoning II
  • Mistakes
  • Lifschitz benchmarks
  • Dialog management

What is there to view?

It is currently not possible to run Alma across the web. Presenting the final state of the KB after the reasoning is not very helpful because the reasoning in Alma depends on the state of the KB at each time step.

Alma however, can write history files that record every addition and deletion to the database at each step. This is what we display here. Accompanying the history in each example is an account of the events of interest in the reasoning.

The history is presented in an applet that allows one to move through the steps and see how the reasoning happened. In this way we preserve the dynamic character of reasoning in Alma.

How to view the examples

The interface is intuitive enough, but a manual is available here.

Some suggestions:

  • The applet will load the history file, and stop at the last step. Click on |< to get to the first step.
  • Click on >> to see the next step.
  • There might be some error messages about the scrollbar. These can be ignored.
